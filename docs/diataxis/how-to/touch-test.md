# タッチ操作テストの使い方

設定画面から起動できる「タッチ操作テスト」画面で、注入された操作が正しく動作しているかを確認できます。

## 起動方法

設定画面 → 「動作確認」→「タッチ操作テストを開く」

## 画面構成と確認方法

### 最終ジェスチャ（注入結果の可視化）

画面上部に「最終ジェスチャ: type / status (detail)」が表示されます。

| フィールド | 説明 |
|-----------|------|
| `type` | ジェスチャの種別（tap / swipe / scroll 等） |
| `status` | `DISPATCHING` / `COMPLETED` / `REJECTED` |
| `detail` | 実行手段（例: `via=CLICK`, `via=SCROLL_UP`） |

### 1) ボタン（タップ/ロングタップ）

「ボタン A/B/C」をタップすると「最終イベント」に `ボタンX: タップ` が表示されます。
長押しが成立すると `ボタンX: 長押し` が表示されます。

### 2) リスト（選択・ダブルタップ）

- 単タップで `リスト: タップ (項目 N)` が表示される
- 長押しで `リスト: 長押し (項目 N)` が表示される
- ダブルタップは `GestureDetector` と同一項目への連続クリックの両方で検知する

### 3) スクロール / スワイプ / ズーム

| 確認項目 | 説明 |
|---------|------|
| 縦スクロール | ListView で上下スクロール |
| 上下左右スワイプ | スワイプ確認エリアで 4 方向のスクロールが成立するか確認 |
| ズーム | ズーム確認エリアでピンチイン/アウトが成立するか確認（倍率 $0.3\times$ 〜 $6.0\times$） |

## 各方式での動作の違い

### スクロール/スワイプキー

| 方式 | 動作 |
|-----|------|
| `ACCESSIBILITY_SERVICE` | 短押し: `ACTION_SCROLL_*` を送出。最終ジェスチャの `detail` が `via=SCROLL_*` または `via=FOCUS_*` になるか確認する |
| `PROXY` | `adb shell input swipe` を送出。スワイプ距離は設定の「スワイプ距離（%）」で調整できる |

`ACTION_SCROLL_*` はジェスチャではなくスクロール命令のため、対象ビューによっては期待どおりに動かない場合があります。

### 「リモコン操作パススルー切り替え」

| 状態 | 説明 |
|-----|------|
| カーソル | 通常のタッチ操作（カーソルは輪っか表示） |
| 方向キー | `DPAD_UP/DOWN/LEFT/RIGHT`・`DPAD_CENTER`・`BACK` が通常操作になる（カーソルは DPAD 風アイコン表示） |

## 「最終イベント」が更新されないとき

以下を確認してください。

- AccessibilityService が有効になっているか
- アプリが前面に表示されているか
- 端末側の制限（特定画面/システム UI への注入ブロック）がないか

## ADB プロキシの反応速度を改善したい場合

`ADB_STATE_POLL_MS` を設定してバックグラウンド監視を有効化すると、入力時の ADB 状態確認をキャッシュから返すため遅延が減ります。
詳細は [プロキシサーバーリファレンス](../reference/proxy-server.md) を参照してください。
