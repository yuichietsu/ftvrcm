# リソース最適化戦略

## 目標
- **CPU使用率**：常時 2% 以下
- **メモリ使用量**：常時 50MB 以下
- **バッテリー消費**：1時間あたり 2% 以下
- **応答性**：リモコン入力から 50ms 以内に反応

## 1. CPU削減戦略

### 1.1 イベント監視の最適化

#### マウス操作モード時のみ処理
```kotlin
// RemoteControlMonitorService で マウス操作モード判定
if (operationMode == OperationMode.MOUSE) {
    // マウスイベント処理のみ実行
    mouseEventProcessor.process(keyEvent)
} else {
    // キーイベントをそのまま通す（処理コストなし）
    return false
}
```

**効果**：通常操作モード時のCPU使用率ほぼ 0%

### 1.2 キーイベント処理の効率化

#### 必要なキーのみ監視
```kotlin
// マウス操作対象キーのみを処理
val mouseKeyList = listOf(
    KeyEvent.KEYCODE_DPAD_UP,
    KeyEvent.KEYCODE_DPAD_DOWN,
    KeyEvent.KEYCODE_DPAD_LEFT,
    KeyEvent.KEYCODE_DPAD_RIGHT,
    KeyEvent.KEYCODE_DPAD_CENTER
)

if (keyCode in mouseKeyList) {
    // 処理対象のキー
} else {
    // 処理対象外 → 即座に return
}
```

**効果**：不要な処理分岐を削減

### 1.3 バッチ処理による連続入力の効率化

#### 複数キー入力を蓄積してから処理
```kotlin
// 10ms ごとに蓄積したキー入力をまとめて処理
val batchProcessor = BatchKeyProcessor(
    batchIntervalMs = 10,
    maxBatchSize = 5
)

keyEvent -> batchProcessor.add(keyEvent)
         -> 蓄積 → 定期処理でまとめて座標計算
```

**効果**：座標計算の回数を 30-50% 削減

## 2. メモリ削減戦略

### 2.1 ViewModel のスコープ管理

#### Activity destroy 時にリソース解放
```kotlin
class SettingsViewModel : ViewModel() {
    private val settingsRepository = SettingsRepository()
    
    override fun onCleared() {
        // ViewModel destroy時に自動呼び出し
        settingsRepository.cleanup()
        observerMap.clear()
    }
}
```

**効果**：メモリリーク防止、GC頻度低減

### 2.2 LiveData の購読管理

#### 不要なオブザーバー削除
```kotlin
class RemoteControlMonitorService : Service() {
    private var modeObserver: Observer<OperationMode>? = null
    
    override fun onDestroy() {
        // Service destroy時にオブザーバー削除
        operationModeLiveData.removeObserver(modeObserver!!)
        modeObserver = null
    }
}
```

**効果**：イベントコールバックによるメモリ蓄積防止

### 2.3 キャッシング戦略

#### SharedPreferences をメモリキャッシュ
```kotlin
// 起動時に SharedPreferences をメモリに読み込む（1回のみ）
val settings = SharedPreferencesDataSource.loadAll()

// その後はメモリキャッシュから読み取り
val mouseSpeed = settings.mouseSpeed
```

**効果**：ファイルI/O削減、レイテンシ低下

### 2.4 リソースリーク検出

#### Lint ルール有効化
- `LeakCanary` をデバッグビルドに組み込み
- メモリリーク検出と通知

## 3. バッテリー削減戦略

### 3.1 ウェイクロック使用制限

#### 不要なウェイクロック排除
```kotlin
// ❌ 避けるべき：ウェイクロック常時保持
val wakeLock = powerManager.newWakeLock(
    PowerManager.PARTIAL_WAKE_LOCK, "ftvrcm::lock"
)
wakeLock.acquire()

// ✅ 推奨：使用時のみウェイクロック
if (isMouseOperationMode && inputEventOccurred) {
    wakeLock.acquire(10) // 10ms のみ保持
}
```

**効果**：バッテリー消費 30-50% 削減

### 3.2 センサー制御の最小化

#### 加速度センサー使用を検討段階で判定
- 初版では加速度センサー不使用
- 将来的に必要に応じて導入検討

### 3.3 位置情報・Bluetooth制御

#### 不要な通信機能の無効化
- GPS 不使用
- Bluetooth スキャン不使用

## 4. ネットワーク削減戦略

### 4.1 ネットワーク通信ゼロ

ftvrcm はローカル処理のみを行い、ネットワーク通信なし

- 設定は SharedPreferences で Local保存
- 更新通知等も不実装

## 5. ストレージ削減戦略

### 5.1 APK サイズ最小化

#### ライブラリ依存を最小化
```gradle
dependencies {
    // 必須のみ
    implementation 'androidx.appcompat:appcompat:1.x.x'
    implementation 'androidx.lifecycle:lifecycle-viewmodel:2.x.x'
    // 不要なライブラリは非導入
}
```

**目標**：APK サイズ 5MB 以下

#### R8/ProGuard による難読化・最適化
```gradle
android {
    buildTypes {
        release {
            minifyEnabled true
            proguardFiles getDefaultProguardFile(
                'proguard-android-optimize.txt'
            ), 'proguard-rules.pro'
        }
    }
}
```

**効果**：APK サイズ 20-30% 削減
