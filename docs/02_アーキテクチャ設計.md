# アーキテクチャ設計

## アーキテクチャパターン：MVVM

ftvrcmは**Model-View-ViewModel（MVVM）**パターンを採用します。

```
┌─────────────┐
│     View    │ (Activity, Fragment, Composable)
└──────┬──────┘
       │ observes
       ↓
┌─────────────┐
│ ViewModel   │ (UI Logic, State Management)
└──────┬──────┘
       │ uses
       ↓
┌─────────────┐
│   Repository│ (Data Layer)
└──────┬──────┘
       │ accesses
       ↓
┌─────────────┐
│ Data Sources│ (SharedPrefs, File, Network)
└─────────────┘
```

## 層別構造

### 1. **Presentation Layer（UI層）**
- **Activity/Fragment**：ユーザーインターフェース
- **SettingsActivity**：設定画面
- **RemoteControlMonitorService**：バックグラウンドリモコン監視

**責務**：UI表示、ユーザー入力受け取り、ViewModel との連携

### 2. **Domain Layer（ビジネスロジック層）**
- **RemoteKeyEvent**：リモコンキーイベント定義
- **RemoteControlMode**：操作モード定義（enum）
- **KeyMapperUseCase**：キーマッピング処理

**責務**：ビジネスロジック、ドメインモデル定義、ユースケース実装

### 3. **Data Layer（データ層）**
- **SettingsRepository**：設定データアクセス
- **SharedPreferencesDataSource**：SharedPreferences ラッパー
- **KeyMappingDataSource**：キーマッピング設定の永続化

**責務**：データの取得・保存、キャッシング、データソース抽象化

## 依存性注入戦略

```
MainActivity/SettingsActivity
    ↓
ViewModel (settingsRepository, keyMapperUseCase)
    ↓
Repository
    ↓
DataSource
```

**方式**：手動DI（軽量性重視）またはHilt（大規模化時）

## バックグラウンドサービス設計

### RemoteControlMonitorService

```
RemoteControlMonitorService (Started Service)
    ↓
KeyboardManager（イベント監視）
    ↓
RemoteEventProcessor
    ├── OperationModeManager（モード切り替え判定）
    └── RemoteEventHandler（イベント処理）
```

**ライフサイクル**：
- 起動：常時（バックグラウンド監視は常に有効）
- 動作：リモコンイベント監視、操作モード判定
- 停止：ユーザー設定で無効化時、またはシステム停止時

## 通信フロー（操作モード切り替え）

```
[リモコンキー入力] 
    ↓
[RemoteControlMonitorService - KeyboardManager]
    ↓
[OperationModeManager - モード判定]
    ↓
YES: マウス操作モード → [MouseEventProcessor]
NO: 通常操作モード → [デフォルト処理]
```

## メモリ管理戦略

### LiveData スコープ管理
- Activity/Fragment が destroy 時に自動アンサブスクライブ
- ViewModel は画面回転時にも保持

### サービス監視最小化
- 必要なイベントのみ購読（マウス操作モード時のみ）
- ウェイクロック使用は最小限

## 拡張性設計
### キーマッピング拡張
キーマッピング設定は JSON 形式で保存、動的にロード可能。
