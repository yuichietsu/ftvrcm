# タッチ操作テスト

設定画面から起動できる「タッチ操作テスト」画面で、注入された操作が端末側で一般的なイベント／UI操作として成立しているかを確認できます。

## 起動方法

- 設定画面 → 「動作確認」→「タッチ操作テストを開く」

## 画面構成と確認ポイント

### 0) 最終ジェスチャ（注入結果の可視化）

- 画面上部に「最終ジェスチャ: type / status (detail)」を表示します。
- `status` は `DISPATCHING` / `COMPLETED` / `REJECTED` を主に使用し、`detail` に実行手段（例: `via=CLICK` / `via=SCROLL_UP`）が入ります。
- 表示は2行固定（はみ出しは省略）です。全文は「最終ジェスチャ」行を選択（クリック）するとダイアログで表示されます。


### 1) ボタン（タップ/ロングタップ）

- 「ボタンA/B/C」を実際にタップ（クリック）すると「最終イベント」に `ボタンX: クリック` が表示されます。
- ロングタップ（長押し）が成立すると `ボタンX: ロングクリック` が表示されます。

### 2) リスト（選択可能・ダブルタップ判定）

- 単タップで項目が選択され、`リスト: クリック (項目 N)` が表示されます。
- ロングタップで `リスト: ロングクリック (項目 N)` が表示されます。
- ダブルタップは、端末の都合で `MotionEvent` が届かないケースでも判定できるように、
  - `GestureDetector` の `onDoubleTap`
  - 同一項目の連続クリック（短時間）
  の両方で検知します。

### 3) スクロール確認（ListView）

- スクロールの確認は ListView の縦スクロールで行います。
- 上下にスクロールすると、クリック対象の項目が変化し、タップ時の表示（`項目 N`）で確認できます。

### 3.1) スワイプ確認（上下左右）

- 画面内に「スワイプ確認（上下左右）」エリアを用意しています。
- 内容物が上下左右にはみ出すようにしてあるため（画面サイズが大きい環境でも縦方向がはみ出すように調整されます）、
  - 縦スクロール（上/下）
  - 横スクロール（左/右）
 それぞれが成立するかを目視で確認できます。

- 背景パターンは方向が分かるように斜線を入れていますが、テストの邪魔にならないように落ち着いた配色にしています（「SWIPE」文字の表示はありません）。

## スワイプが思った向きに動かない/反応しないとき

- 設定の「エミュレーション方法」により、上下左右キーの注入方法が変わります。
  - `アクセシビリティサービス`: アクセシビリティのスクロール操作（`ACTION_SCROLL_*`）を利用します。
  - `PROXY`: PC上プロキシ経由で `adb shell input swipe` によるスワイプを利用します。
- `ACTION_SCROLL_*` は「ジェスチャとしてのスワイプ」ではなく「スクロール命令」なので、対象ビューによっては期待どおりに動かない場合があります。
- `アクセシビリティサービス` の場合、「最終ジェスチャ」が `scroll_up` / `scroll_down` の `COMPLETED` になるかで、スクロール命令が成立しているか確認できます。

### 「上下左右スクロール」キーの動作

- `アクセシビリティサービス` の場合
  - 短押し: `ACTION_SCROLL_UP/DOWN/LEFT/RIGHT` を送出します（`UP/DOWN`は環境によりRejectされるため、内部で`BACKWARD/FORWARD`も試行します）。
  - 長押し: 設定で「長押しで連続実行」がONの場合、一定間隔で連続実行します。
  - 「最終ジェスチャ」の `detail` で、`via=SCROLL_*`（スクロール系）/ `via=FOCUS_*`（DPAD系）になっているか確認できます。

- `PROXY` の場合
  - `adb shell input swipe` を送出します（最終ジェスチャの表示も更新されます）。
  - スワイプはカーソル位置を開始点として送出します。
  - スワイプ距離は設定の「スワイプ距離（%）」で調整できます。

### 「カーソル/方向キー切り替え」

- 「カーソル」: 現状のタッチ操作です（カーソルは輪っか）。
- 「方向キー」: `DPAD_UP/DOWN/LEFT/RIGHT` と `DPAD_CENTER` だけ通常操作になります（サービスが消費しません）。このときカーソルはDPAD風アイコンになります。

## 「最終イベント」が更新されないとき

- アクセシビリティサービスが有効か
- アプリが前面に表示されているか
- 端末側の制限（特定画面/システムUIへの注入ブロック等）がないか

を確認してください。
