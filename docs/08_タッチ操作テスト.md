# タッチ操作テスト

設定画面から起動できる「タッチ操作テスト」画面で、注入された操作が端末側で一般的なイベント／UI操作として成立しているかを確認できます。

## 起動方法

- 設定画面 → 「動作確認」→「タッチ操作テストを開く」

## 画面構成と確認ポイント

### 0) 最終ジェスチャ（注入結果の可視化）

- 画面上部に「最終ジェスチャ: type / status (detail)」を表示します。
- `status` は `DISPATCHING` / `COMPLETED` / `CANCELLED` / `REJECTED` で、`detail` にフォールバック有無（例: `fallback=SCROLL_VERTICAL ok`）が入ります。


### 1) ボタン（タップ/ロングタップ）

- 「ボタンA/B/C」を実際にタップ（クリック）すると「最終イベント」に `ボタンX: クリック` が表示されます。
- ロングタップ（長押し）が成立すると `ボタンX: ロングクリック` が表示されます。

### 2) リスト（選択可能・ダブルタップ判定）

- 単タップで項目が選択され、`リスト: クリック (項目 N)` が表示されます。
- ロングタップで `リスト: ロングクリック (項目 N)` が表示されます。
- ダブルタップは、端末の都合で `MotionEvent` が届かないケースでも判定できるように、
  - `GestureDetector` の `onDoubleTap`
  - 同一項目の連続クリック（短時間）
  の両方で検知します。

### 3) スクロール領域（4色パターン・縦横スクロール）

- 4色のパターン領域を上下にスワイプしてスクロールすると、`縦スクロール: y=...` が表示されます。
- 左右にスワイプしてスクロールすると、`横スクロール: x=...` が表示されます。

## スワイプが思った向きに動かない/反応しないとき

- 本アプリは基本的に「画面上のスワイプ（ジェスチャ注入）」を実行します。
- ただし、端末や画面によってはジェスチャ注入がキャンセルされることがあり、その場合はフォールバックとしてアクセシビリティのスクロール操作（`ACTION_SCROLL_*`）を実行します。
  - `ACTION_SCROLL_*` は「ジェスチャとしてのスワイプ」ではなく「スクロール命令」なので、対象ビューによっては縦/横の解釈が異なることがあります。
- 左右スワイプが無反応にならないよう、フォールバックは縦方向だけでなく横方向（LEFT/RIGHT）も試行します。

フォールバックが有効な場合、ジェスチャ注入がキャンセルされたときに「最終ジェスチャ」が `CANCELLED (fallback=... ok/failed)` のように更新されます。
フォールバックが無効な場合は `CANCELLED (fallback=DISABLED)` のように表示されます。

## 「最終イベント」が更新されないとき

- アクセシビリティサービスが有効か
- アプリが前面に表示されているか
- 端末側の制限（特定画面/システムUIへの注入ブロック等）がないか

を確認してください。
