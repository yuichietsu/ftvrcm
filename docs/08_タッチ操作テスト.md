# タッチ操作テスト

設定画面から起動できる「タッチ操作テスト」画面で、注入された操作が端末側で一般的なイベント／UI操作として成立しているかを確認できます。

## 起動方法

- 設定画面 → 「動作確認」→「タッチ操作テストを開く」

## 画面構成と確認ポイント

### 1) ボタン（タップ/ロングタップ）

- 「ボタンA/B/C」を実際にタップ（クリック）すると「最終イベント」に `ボタンX: クリック` が表示されます。
- ロングタップ（長押し）が成立すると `ボタンX: ロングクリック` が表示されます。

※ 端末側でジェスチャ注入がキャンセルされた場合、フォールバックとしてノード操作（`ACTION_CLICK` / `ACTION_LONG_CLICK`）が実行されることがあります。
この場合でもボタンのクリックは発火し得るため、「UI操作として届いているか」の切り分けに使えます。

### 2) リスト（選択可能・ダブルタップ判定）

- 単タップで項目が選択され、`リスト: クリック (項目 N)` が表示されます。
- ロングタップで `リスト: ロングクリック (項目 N)` が表示されます。
- ダブルタップ（短時間に2回タップ）で `リスト: ダブルタップ (項目 N)` が表示され、該当項目を選択します。

### 3) スクロール領域（模様付き）

- 縞模様（「模様 1..」）の領域を上下にスワイプしてスクロールすると、`スクロール: y=...` が表示されます。
- 速めにスワイプした場合は、方向判定として `スクロール領域: 上スワイプ/下スワイプ/左スワイプ/右スワイプ` が表示されることがあります。

## 「最終イベント」が更新されないとき

- アクセシビリティサービスが有効か
- アプリが前面に表示されているか
- 端末側の制限（特定画面/システムUIへの注入ブロック等）がないか

を確認してください。
