# トラブルシュート

## 1. アクセシビリティ設定にサービスが出ない（Fire OS）

Fire OSのビルドや端末設定によっては、設定UIから「ダウンロードしたサービス（Accessibility Service）」のトグルが見つからないことがあります。
この場合でも、ADBが使えるならシェルから有効化できることがあります。

1. Fire TVで`開発者オプション` → `ADBデバッグ`をON
2. PCから接続（同一LAN）
   - `adb connect <FIRE_TV_IP>:5555`
3. サービス有効化（コンポーネント名）
   - `adb shell settings put secure enabled_accessibility_services com.ftvrcm/com.ftvrcm.service.RemoteControlAccessibilityService`
   - `adb shell settings put secure accessibility_enabled 1`
4. 反映しない場合は再起動
   - `adb reboot`

有効化後、アプリの設定画面「デバッグ → サービス状態」が`接続: ...`になれば接続できています。

## 2. ポインタは出るがクリックが発生しない

### 2.1 まず確認すること

- マウスモードに入ると、緑色のポインタ（overlay）が表示されます。
- クリックは、`AccessibilityService`からの注入（ジェスチャまたはノードクリック）で行います。

設定画面の「デバッグ → ジェスチャ注入状態」を確認してください。

### 2.2 表示の読み方

- `tap#.. completed ...`
  - OSがジェスチャ注入を完了しました。クリックしない場合は、座標ずれやクリック対象が特殊（システムUI等）の可能性があります。

- `tap#.. rejected (dispatchGesture=false)`
  - OSがジェスチャ注入を開始すら受け付けませんでした。端末側制限の可能性があります。

- `tap#.. cancelled ...`
  - OSがジェスチャ注入を途中でキャンセルしました。Fire OSで起きることがあります。
  - 本プロジェクトではフォールバックとして、カーソル座標上の`AccessibilityNodeInfo`を探索して`ACTION_CLICK`を試みます。

- `tap#.. cancelled -> nodeClick OK`
  - ノードクリックでのクリック実行に成功しました。

- `tap#.. cancelled -> nodeClick FAIL`
  - クリック対象ノードが見つからない / クリック不可 / システムUIで制限、などが考えられます。

### 2.3 うまくいかない場合

- Fire OS側で、特定アプリやホーム画面（システムUI）への注入が制限されている場合があります。
- 「デバッグ → 権限/能力チェック」の `cap.gestures` が `NO` の場合、ジェスチャ注入は期待できません。

## 3. 再生ボタン2度押しでモード切替（実験）が他アプリ中に効かない

Fire OSがメディアキー（Play/Pause）をグローバルに第三者へ配送しない構成の場合、設定画面（フォーカス中）では拾えても、ホーム画面/他アプリ中では拾えません。
この場合はアクセシビリティサービスの「キー監視（filterKeyEvents）」経路でのトグルを使ってください。
