# 機能仕様書

## 1. 操作モード切り替え機能

### 1.1 操作モードの種類

| モード | 説明 | リモコン動作 |
|--------|------|------------|
| **通常操作** | テレビリモコンの標準動作 | 上下左右、決定、戻る等が通常通り |
| **タッチ操作** | ポインタ移動とタップ等を提供 | 特定キーでポインタ移動、タップ等 |

### 1.2 モード切り替え方法

#### 方法A：専用キーで切り替え
- リモコンの特定キー（デフォルト：BACKキー 長押し）で即座に切り替え
- 他アプリ実行中でも無条件に切り替え可能
- バックグラウンドサービスで常時監視

#### 方法B：設定アプリから手動切り替え
- ftvrcmアプリを開き、設定画面から選択
- 変更後、設定画面終了時に即座に反映

### 1.3 モード切り替え時の動作

```
[切り替え操作]
    ↓
[RemoteControlMonitorService で検知]
    ↓
[モード状態を SharedPreferences に保存]
    ↓
[UI 更新（表示モード表示）]
    ↓
[次のリモコン入力から新モード適用]
```

## 2. タッチ操作機能

### 2.1 タッチ操作に割り当て可能なキー

| リモコンキー | タッチ操作 | デフォルト割り当て |
|------------|---------|----------------|
| **十字キー上** | ポインタ上移動 | ✓ |
| **十字キー下** | ポインタ下移動 | ✓ |
| **十字キー左** | ポインタ左移動 | ✓ |
| **十字キー右** | ポインタ右移動 | ✓ |
| **決定ボタン** | タップ（短押し）/ロングタップ（長押し） | ✓ |
| **CH+ / CH-** | 上下スワイプ | ✓ |
| **REW / FF** | 左右スワイプ | ✓ |
| **その他** | カスタム割り当て可能 | ✗ |

補足：スワイプは、現在のカーソル位置から開始します。
### 2.2 ポインタ移動のパラメータ

- **移動速度**：1ステップあたりのポインタ移動量（デフォルト10）
- **連続移動**：方向キー押しっぱなしで一定間隔で移動し続ける
- **加速度**：方向キー押しっぱなしで徐々に加速（常に有効）
- **感度調整**：設定画面から変更可能

### 2.3 タッチ操作の実装

```
[リモコンキー入力（タッチ操作モード）]
    ↓
[KeyboardManager - KeyEvent キャッチ]
    ↓
[MouseEventProcessor - 座標計算]
    ↓
[MotionEvent 生成 → ACTION_MOVE または ACTION_DOWN]
    ↓
[InputMethodManager 経由で入力イベント送信]
    ↓
[他アプリがタッチ操作を受け取り]
```

## 3. 設定画面UI

### 4.1 設定画面の構成

```
[ftvrcm 設定]

【操作モード】
- 現在のモード: タッチ操作 / 通常操作
- モード切り替えキー: [BACK]
- 長押しで切り替え: [有効 / 無効]

【タッチ操作設定】
- 移動量: [スライダー]
- タッチ操作開始時のカーソル位置: [中央/四隅/前回の位置...]
- エミュレーション方法: [Accessibility Service / ADB]
- 各操作のキー割当
    - 決定ボタン → タップ（短押し）/ロングタップ（長押し）
    - CH+ / CH- → 上下スクロール（`Accessibility Service`）/ 上下スワイプ（`ADB`）
    - REW / FF → 左右スクロール（`Accessibility Service`）/ 左右スワイプ（`ADB`）

【システム】
- 必要状態：アクセシビリティサービス: ON / OFF
- バックグラウンド監視: 有効 / 無効
- アクセシビリティ設定を開く
- アプリ情報を開く
- デフォルト設定に戻す（確認あり）
```

### 3.2 設定画面のナビゲーション

- タブ / 折りたたみパネルで各設定カテゴリを整理
- Fire TV Stick のリモコン十字キーで操作可能
- 十字キー下で次の項目へ移動
- 決定ボタンで値を変更

## 4. バックグラウンド動作

### 4.1 バックグラウンドサービス

**RemoteControlMonitorService** が常時リモコンイベント監視

```
[Service 起動]
    ↓
[KeyboardManager - android.view.KeyEvent 監視開始]
    ↓
[リモコンキー入力 → KeyEvent キャッチ]
    ↓
[OperationModeManager でモード判定]
    ↓
タッチ操作モード → MouseEventProcessor
通常操作モード → デフォルト処理（キー通過）
```

### 4.2 バックグラウンド起動タイミング

- **起動時**：ftvrcm 初回起動時、または「バックグラウンド監視有効」に変更時
- **再開時**：デバイス起動時（BOOT_COMPLETED インテント）
- **停止時**：「バックグラウンド監視無効」に変更時

### 4.3 メモリ・CPU 削減策

- **イベント フィルタリング**：タッチ操作モード時のみ処理
- **遅延処理**：連続キー入力を蓄積して処理
- **アンバウンディング**：ViewModel 破棄時に自動解放

## 5. リモコンキーイベント

### 5.1 対応リモコン

Fire TV Stick の標準リモコン：
- 十字キー（上下左右）
- 決定ボタン
- 戻るボタン
- ホームボタン（HOMEキー）
- その他ボタン（オプション）

### 5.2 キーイベント処理フロー

```
[KeyEvent.KEYCODE_* ↓ KeyboardManager]
  → [RemoteKeyMapper 変換]
    → [モード判定]
        → [タッチ操作 / 通常操作]
```
